Password Security Analyzer

Directed Study Project by Ashley Zapata

Try the Analyzer

Project Overview

This project is a web-based password analyzer designed to help users understand whether a password is secure. It evaluates password strength based on length, character variety, predictable patterns, and breach exposure.

The website combines cybersecurity concepts with full-stack development. The user enters a password, the backend analyzes it, and the results are displayed instantly as a score, strength level, issue list, and breach-exposure warning.

The goal of the project is to make password security easier to understand in a practical, interactive way.

Why I Chose This Project

I chose this project because password security is one of the most common weaknesses in cybersecurity. Many people still create passwords that are short, predictable, reused, or already exposed in breach data.

I wanted to build something practical that connects directly to real-world security problems. Instead of making a static informational project, I wanted to create a tool that actually lets users test and understand password risk.

Over time, the project expanded from a simple analyzer into a full website with a live demo, breach checking, and a more polished user experience.

Features

Core Features

  • Password strength classification
  • Numeric score from 0 to 100
  • Checks for uppercase, lowercase, numbers, and special characters
  • Length validation
  • Common password detection

Advanced Features

  • Strength meter bar
  • Show / hide password eye icon
  • Breach exposure check using known breach datasets
  • Real-time issue reporting
  • Interactive website interface using Flask

Challenges I Faced

One challenge I faced was learning how to connect the frontend website to backend analysis logic. I had to understand how Flask handles routes, templates, and form submission.

Another challenge was implementing the breach-exposure feature in a privacy-conscious way. I had to learn how SHA-1 hashing and k-anonymity work so the password would not be directly sent in plain text to the breach-check system.

I also had to organize the folder structure correctly, debug file paths, and make sure the project was large enough in scope to work as a semester-long directed study.

What I Learned

This project taught me how a cybersecurity tool can be built incrementally from a basic script into a more complete product.

I learned how frontend and backend systems communicate, how password analysis logic can be structured, and how external security data can be integrated into a project.

I also learned more about hashing, breach datasets, debugging, and how to present technical work in a way that is understandable to users.
